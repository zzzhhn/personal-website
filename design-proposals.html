Design System Review

个人网站设计风格优化提案

基于对你当前 Astro + React + Tailwind 项目的完整审读，从排版系统、色彩架构、布局节奏、交互层次、组件语言五个维度，提出 7 套可落地的设计方向。每套方案附带 mock 渲染与推荐理由。

Current State Diagnosis / 现状诊断

你的设计做对了什么，以及哪里可以更好

STRENGTH

Liquid Glass 体系自洽

你的 glass / glass-hover / glass-subtle / glass-elevated 四级毛玻璃层级清晰，specular highlight 和 ambient mesh background 制造了不错的空间深度。这是一个有「意图」的设计系统。

CRITICAL

字体层次单一

全站仅 Geist Sans 一种字体，三个字重（400/500/600）。标题与正文的区分完全依赖 font-size + font-weight，缺乏「质感对比」。Quantitative Finance × AI 的定位需要一种更有权威感的 display typeface 来支撑专业形象。

IMPORTANT

Section Heading 过度一致

所有 section heading 使用完全相同的模式：居中文字 + 2rem 宽的 accent 色短横线。重复 6 次后产生视觉疲劳（About → Education → Experience → Projects → Campus → Contact），且没有引导阅读节奏的变化。

IMPORTANT

Hero CTA 密度过高

Hero 区域纵向堆叠了 5 个 glass 按钮（Resume / GitHub / LinkedIn / Email），每个都有 icon + label + detail。信息量对等导致没有明确的 primary action，且占据了大量 hero 黄金空间，削弱了 "第一印象" 的冲击力。

STRENGTH

Dark/Light 双模式完备

oklch 色彩空间的运用很现代，且 light mode 下 bokeh 粒子和 dark mode 下星座粒子的区分说明你认真考虑了双模式的氛围差异。

CRITICAL

色彩表达单调

除了 accent blue (oklch 0.68 0.16 255) 之外，几乎没有任何辅助色参与表达。Status badge 的绿色/黄色是唯一例外。整站缺乏色彩节奏，"premium" 感有但 "unique" 感不足。

Design Proposals / 设计方案
01

引入 Serif Display 字体，建立「权威 × 精致」的排版对比

Typography Contrast System — 用衬线体做标题，无衬线体做正文，Mono 做 metadata

Haonan Zhong
Quantitative Finance × Artificial Intelligence
Qf 衬线体天然携带「学术」「金融」「权威」的隐性联想。一个 Instrument Serif 标题搭配 Inter/Geist 正文，瞬间将视觉等级拉开两档。
推荐理由

为什么这对你特别重要

你的定位是 "Quantitative Finance × AI"——这两个领域分别对应金融的庄重感和科技的锐利感。单一的 Geist Sans 只能表达后者。引入一款高质量 Serif（推荐 Instrument Serif、Fraunces 或 Newsreader）做 display 用途，能同时传递两种信号。

具体做法

  • 所有 section h2、hero name 改用 Serif Display，font-weight 400，letter-spacing -0.02em
  • 正文保留 Geist Sans 不变，形成「优雅标题 + 清晰正文」的经典双字体配对
  • Timeline 日期、status badge、tech tag 使用 JetBrains Mono 或 Geist Mono，强调 "数据/代码" 属性
  • 三层字体系统：Serif (Display) → Sans (Body) → Mono (Data)，每层职责清晰

预期效果

网站从 "好看的技术博客" 升级为 "有辨识度的个人品牌"。衬线标题 + 无衬线正文的对比在 Stripe、Linear、Vercel 等顶级网站中被反复验证有效。

推荐理由

当前问题

你的 accent 只有蓝色一种，所有高亮元素（section 横线、timeline dot、tech category label、CTA glow）全部是同一个 oklch(0.68 0.16 255)。色彩单一导致视觉上 "平"，缺乏节奏变化。

具体做法

  • 保留蓝色作为 primary accent（链接、CTA、active 状态）
  • 新增 warm amber oklch(0.75 0.14 55) 作为 secondary accent，用于 status badge "In Progress"、奖项高亮、hover glow 等「动态/进行时」语境
  • 新增 teal oklch(0.72 0.12 165) 作为 tertiary accent，用于 "Completed" badge、证书标记等「确定/已完成」语境
  • 三种色彩携带语义：蓝=行动、暖=进行、青绿=完成

预期效果

每种颜色出现时用户潜意识里知道它在传递什么。色彩从「装饰」升级为「信息通道」，整站的 premium 感和 unique 感同时提升。

03

重构 Hero 信息层级——聚焦一个 Primary CTA

Hero Hierarchy — 把 5 个等权按钮缩减为 1 个主按钮 + 1 组 icon links

Portrait
Haonan Zhong
Quantitative Finance × AI
↑ 一个实心主按钮 + 三个幽灵按钮，inline 排列
推荐理由

当前问题

5 个纵向堆叠的 glass card 按钮，每个都有 icon + label + detail text，总高度超过 300px。用户到达 hero 后视觉重心被 5 个等权元素分散，无法直觉判断 "你最希望我做什么"。

具体做法

  • Resume 按钮作为唯一 primary CTA：实心填充 + accent 色，体积最大
  • GitHub / LinkedIn / Email 降为 ghost button 或纯 icon button，水平排列在 primary CTA 旁边
  • 去掉每个按钮的 detail 行（如 "github.com/zzzhhn"），这些信息在 Contact section 重复出现
  • 按钮区从纵向 stack 改为横向 inline-flex，释放 200px+ 垂直空间给 tagline 或一句 bio summary

预期效果

Hero 的信噪比显著提升。访客 3 秒内完成 "这是谁 → 他做什么 → 我可以下载简历" 的完整认知链路。Minimalist 的核心就是每个元素都不可再删。

推荐理由

当前问题

6 个 section 使用完全相同的 heading 模式（居中 sans-serif + 2rem accent 短横线），长页面滚动时产生 "每个板块看起来都一样" 的既视感。Premium 网站的特征之一是 controlled variety——在统一中有变化。

具体做法

  • 定义 2-3 种 heading 变体，按照 section 的内容特性匹配： 「Editorial 式」适合 About / Education 等叙事性 section； 「Numbered + Line」适合 Experience / Projects 等列表性 section； 「居中+短横」保留给 Contact 等简短 section
  • 添加 section 编号（01 / 02 / 03...），既提供导航锚点感，也增加 editorial 质感
  • heading 左对齐 + section 内容居中的不对称布局，制造视觉张力

预期效果

每到一个新 section，用户感到 "节奏变了"，注意力被重新唤起。同时 section 编号制造了 "有组织的叙事" 感，符合金融/量化背景的 "结构化思维" 人设。

05

丰富卡片语言——不是所有容器都要 Glassmorphism

Card Vocabulary — 引入 outline / solid / accent-border 等变体，降低 glass 的视觉通胀

Glass（现状）

适合 hero CTA、header 等需要「悬浮」感的元素

Outline

适合 tech stack 分类、awards 等信息密度高的区块

Solid

适合 project card 等需要视觉重量的主内容卡片

Accent Line

适合 timeline entry、highlight 等需要引导视线的元素

推荐理由

当前问题

目前几乎所有内容容器都使用 .glass class：education card、awards、tech categories、timeline card、project card、certificates...当「毛玻璃」成为默认，它就失去了 "特殊" 的语义。就像如果每句话都加粗，加粗就等于没加粗。

具体做法

  • Glass 保留给最高层级：header、hero CTA、modal overlay
  • Project Card 改为 solid background（更重的视觉权重，暗示 "这是重要内容"）
  • Tech Category / Awards 改为 outline style（轻量、通透）
  • Timeline Entry 改为 accent-border（左边框引导视线方向）
  • 整体原则：层级越高 → 效果越丰富（glass），层级越低 → 越轻量（outline/solid）

预期效果

不同容器承载不同的视觉权重，用户在 scroll 过程中自然感知到 "这个更重要" vs "这是辅助信息"。Glassmorphism 因为稀缺而重新变得 premium。

升级微交互——从「hover 浮起」到「有意义的反馈」

Cursor Proximity Effect：Card 边框随鼠标接近渐亮（类似 Raycast.com），用 JS 监听 mousemove，计算 cursor 到 card 的距离，动态设置 border-color opacity。

Staggered Scroll Reveal：当前的 fade-up 全部同时出现。改为每个 card 延迟 80-120ms 依次淋入，制造 "瀑布流" 般的节奏感。你的 CSS 里已有 --stagger-index 变量但未充分利用。
推荐理由

当前交互层次

你的 glass-hover 效果是 translateY(-3px) + scale(1.005) + shadow 加深。这个组合很「标准」，但不够 memorable。premium 网站的交互特征是让用户感觉 "这个网站在回应我的存在"。

具体做法

  • Cursor Proximity Glow：卡片边框根据鼠标距离 0-200px 渐变发光，参考 Raycast / Vercel 的卡片效果。纯 JS + CSS custom property 实现，不需要额外库
  • Project Card hover 时显示一个微妙的 gradient overlay 扫过表面（光影掠过感），而非简单浮起
  • Timeline Card 展开动画从 height:0 硬切改为 spring 缓动 + opacity 渐入
  • Nav link hover 加入 underline 从左到右 "生长" 的动画，替代纯背景色变化

预期效果

用户感受到 "这个网站有活力"，而不仅是 "这个网站好看"。交互从 "状态标识" 升级为 "体验设计"。

推荐理由

Timeline 重设计

  • 当前日期信息嵌在 card 内部右上角的 badge 里，视觉权重低于 role/org，但时间线最重要的信息恰恰是时间
  • 将日期独立为左侧列（mono 字体，右对齐），与卡片形成三列布局：date | dot | card。时间维度一目了然
  • 不同经历类型（full-time / internship / research）用不同颜色的 dot 区分，对应 Proposal 02 的三色系统
  • Timeline 的连接线从 gradient fade 改为虚线（更精致）或保持但收窄到 0.5px（更极简）

Header 导航

  • 当前 nav-pill hover 只是背景色变化。建议加入 scroll-spy 机制，当前 section 对应的 nav item 自动获得 active 态（pill background），形成「位置感」
  • Logo "HZ" 的 gradient 文字效果过于微妙（gradient clip 在小字号上几乎看不出）。建议改为纯色 + 加粗，或者换成 monospace 风格的 "HZ." 更有辨识度

## Background

The first pass was a "television": 27 GET endpoints, zero user actions. The refactor (REFACTOR_PLAN W1 / W2) turned it into a research workstation where the LLM is a controlled tool inside the loop, not a black box around it.

## Architecture

Two halves connected by HTTP:

1. **Backend** — FastAPI on an AutoDL RTX 5090 node, tunnelled via Cloudflare. Pydantic schemas, AST-based safe evaluation, a numpy-only vectorized operator layer, per-router `/healthz/routers` endpoint. Kimi For Coding serves the LLM over an Anthropic-compatible `/messages` API.
2. **Frontend** — Next.js 16 App Router on Vercel. Interactive dashboards for data, alpha signals, backtests, factors, gate editing, stress tests. zh / en toggle, dark / light theme, accent-color semantics for IC magnitude.

## Hypothesis Translator (T1)

One sentence in, an executable FactorSpec out:

- System prompt enumerates the 15-op whitelist, operand set, and one worked example
- LLM output parsed as Pydantic `FactorSpec` (name, hypothesis, expression, operators_used, lookback, universe, justification)
- AST walker over `ast.parse(mode="eval")` rejects attribute access, imports, lambdas, non-numeric constants, keyword args; enforces `declared_ops == used_ops` equality so the LLM cannot silently drop or sneak in operators
- 20-ticker geometric-Brownian-motion panel of (lookback + 11 days) is generated on the fly; per-day cross-sectional Spearman IC over the last 10 days, averaged

End-to-end: ~3 s per hypothesis via Kimi (667 tokens average), smoke in 2–5 ms.

## Vectorized Scan Layer

Fifteen pure-numpy operators on a `(T, N)` panel:

- Time-series (axis=0): `ts_mean`, `ts_std`, `ts_zscore`, `ts_rank`, `ts_corr`
- Cross-section (axis=1): `rank` (percentile), `scale` (demean + L1 norm), `winsorize`
- Elementwise: `log`, `sign`, `add`, `sub`, `mul`, `div` (NaN on zero divide), `pow_`

All NaN-safe, all pure, dispatched by an AST walker — not `eval`.

## Key Results

- Every factor that reaches the real backtest engine is already proven to be numerically executable on a synthetic panel
- Distinct HTTP codes (503 no LLM, 502 LLM error, 422 schema / AST fail, 500 smoke crash) so the UI surfaces precise failures instead of a blanket "Network error"
- `/healthz/routers` exposes which FastAPI routers loaded at startup, catching missing-dependency 404s that would otherwise hide behind a 200-OK homepage

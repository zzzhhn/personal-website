## Background

Entry for the Tencent Music AI Hackathon 2026 ("Build for Music+", Track B). QQ Music's Automix can crossfade between songs, but it has no opinion about *which* song should come next — 合拍 is the song-selection brain: it analyzes what makes two tracks mixable and picks the next song the way a DJ would.

## How It Works

1. **Analysis** — librosa extracts BPM, musical key (mapped onto the Camelot wheel), and an energy profile for every track in the library
2. **Matching** — given a seed song, candidates are scored by harmonic compatibility (adjacent Camelot positions), tempo distance, and energy continuity; the Top-5 are surfaced as mix candidates
3. **Mixing** — a beat-aligned crossfade engine schedules the transition on beat boundaries, so consecutive tracks blend like a live DJ set rather than a hard fade
4. **Modes** — four listening modes trade off energy arc and variety (steady groove vs. build-up sets)

## Engineering

Python + FastAPI backend for analysis and recommendation, vanilla JS front end. 53 Python tests cover feature extraction, Camelot compatibility scoring, and the transition scheduler. AI-generated cover art and an audio-quality iteration loop rounded out the MVP.

## Live Demo

The public demo is a fully static build: the pairing and sequencing math is ported 1:1 to JavaScript (verified against the Python implementation with 41 golden test cases producing byte-identical output), and the mix itself plays through a dual-deck Web Audio pipeline — equal-power crossfades at precomputed cue points, plus a BiquadFilter bass-swap for DJ mode — instead of waiting on server-side rendering. The demo library uses Kevin MacLeod's CC-BY 4.0 catalog so the whole thing can live on the open web.
